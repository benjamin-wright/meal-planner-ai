## Documentation

- All sub-documents navigable from `README.md`
- Use clear and concise language suitable for both technical and non-technical stakeholders.
- Include diagrams where appropriate to illustrate architectural concepts.
- Ensure that all documentation is well-organized and easy to navigate.
- Segregate documentation applicable to different personas, to reduce cognitive load for human users and provide more focused context for AI actors
- Demonstrate example data structures using simple JSON where appropriate, but avoid using example code snippets.
- Avoid duplicating information already covered in the standards.
